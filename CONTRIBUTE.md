# 🤝 **Contributing to Number Classification API**  

Thank you for your interest in contributing to the **Number Classification API**! 🎉  
We welcome contributions to improve functionality, fix bugs, or enhance documentation.  

---

## 🛠 **How to Contribute**
### **1️⃣ Fork the Repository**
Click the **Fork** button at the top-right of this repository to create your own copy.  

### **2️⃣ Clone Your Forked Repository**
```bash
git clone https://github.com/your-username/number-classification-api.git
cd number-classification-api
```

### **3️⃣ Create a New Branch**
```bash
git checkout -b feature-name
```
Use a meaningful branch name, like `feature-enhanced-validation` or `bugfix-404-error`.  

### **4️⃣ Install Dependencies**
```bash
pip install -r requirements.txt
```

### **5️⃣ Make Your Changes**
Modify the **FastAPI application (`main.py`)** or other necessary files.  

### **6️⃣ Run & Test Locally**
```bash
uvicorn main:app --reload
```
Test using **Swagger UI**:  
📌 Open: [http://localhost:8000/docs](http://localhost:8000/docs)  

### **7️⃣ Commit Your Changes**
```bash
git add .
git commit -m "Added feature XYZ"
```

### **8️⃣ Push to Your Fork**
```bash
git push origin feature-name
```

### **9️⃣ Submit a Pull Request**
- Go to **your GitHub fork**  
- Click **Compare & Pull Request**  
- Write a brief description of your changes  
- Submit your **Pull Request (PR)**  

---

## ✅ **Contribution Guidelines**
🔹 Follow the **PEP 8** coding style  
🔹 Keep commits **small and meaningful**  
🔹 Add **comments** where necessary  
🔹 Write **clear commit messages**  
🔹 Test before submitting a PR
